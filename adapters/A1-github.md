# A1 — GitHub Adapter

## Use when

Activate when the project uses GitHub for repository hosting, issue tracking, pull requests, or automation.

## Operationalizes

- change management
- problem tracking
- dependency management
- documentation and discoverability

## Adds

- issue and pull request usage conventions
- how verification evidence appears in PR context
- how problem tracking maps onto GitHub issues or labels
- how decision and dependency references are linked from repository workflows

## Does not define

- universal branch strategy
- mandatory label set for every team
- one required PR template for all projects
