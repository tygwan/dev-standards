# P1 — Research / Data / ML Profile

## Use when

Activate this profile when project outcomes depend heavily on:

- experiments
- benchmarks
- model evaluation
- dataset integrity
- reproducibility-heavy analytical claims

## Purpose

This profile strengthens evidence, reproducibility, and artifact traceability beyond the universal core.

## Adds on top of core

- experiment records for meaningful comparisons
- benchmark input and metric disclosure
- artifact lineage for important outputs
- stronger evidence expectations for findings or claims
- reproducible analysis or audit scripts where results depend on data transformations

## Why this is not core

These practices are valuable, but they are too heavy for many projects that do not depend on experiment quality or dataset trust.

## Typical activation signals

- evaluation metrics drive decisions
- processed data is a product or critical asset
- models, prompts, or retrieval systems are benchmarked
- teams need to re-derive numerical claims later

## Typical non-goals

- forcing notebooks, charts, or audit scripts into every project
- forcing model-specific process on non-model systems
