# Profiles

Profiles are optional practice packs for recurring project shapes.

They should:

- add stricter guidance than the core
- activate only when relevant
- avoid leaking topic-specific requirements into the universal core

## Initial Profile Catalog

- `research-data-ml`
- `ai-assisted-development`
- `incident-operations`
- `public-writing`

## Note On AI-Assisted Development

The AI-assisted profile is the correct home for:

- scoped work contexts
- project memory practices
- selective retrieval guidance
- human checkpoints for structural AI-assisted changes

## Authoring Standard

Each profile should answer:

- when to activate it
- what risks or obligations it addresses
- what it adds on top of the core
- why it is not universal
